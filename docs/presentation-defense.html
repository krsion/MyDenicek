The Problem

Denicek is a document-oriented programming system — users program by editing tree-structured documents.

  • Documents = tagged trees (records, lists, references)
  • Addressed by paths: speakers/0/name
  • Users record & replay edit sequences
  • But collaboration uses fragile OT

Goal: replace OT with a robust CRDT that preserves user intent.

Why Not Existing CRDTs?

Automerge

Wrap = delete + re-insert (not atomic).
Concurrent wraps → node under two parents.

Loro

Opaque node IDs → can't express ../../0/source.
Recorded edits can't retarget after structural changes.

⬇
mydenicek: paths are the native addressing mode. OT operates directly on them.

How It Works

  1. Each edit creates an event in a causal DAG (grow-only set = CRDT)
  2. Events sync to peers via WebSocket relay
  3. To get the document: topological sort → replay → transform concurrent selectors
  4. Same events → same order → same transforms → same document on all peers

The hard part isn't convergence (that's free from G-Set + deterministic eval).
The hard part is intention preservation — each edit should do what the user meant.
